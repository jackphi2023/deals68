# Deals68.com Production Beta — React/Vite/Supabase/Netlify

This repository converts the Claude Design V1.6.2 prototype into a production-oriented React/Vite web app for `deals68.com`.

## Stack

- React 19 + Vite
- Supabase Auth, PostgreSQL, RLS, Storage
- Netlify SPA deploy
- Seed scripts for Admin, 6 business deals, 624 imported investors, promo codes, quality criteria

## Key implemented flows

### Public pages

- Home hero search for Businesses and Investors
- Business listing with public filters/counts
- Investor listing with public masked profiles
- Business detail with investor-only detailed Business Quality Score
- Investor detail with hidden private contact
- Pricing page with one Business column; Standard/Featured is selected in Business registration
- Valuation calculator

### Business flow

- Business registration with Standard/Featured plan selection
- Business dashboard profile editing
- Sensitive edits are written to `pending_changes_json`; public profile is not updated until Admin approval
- File upload: Word, Excel, PPT, PDF, CSV into Supabase Storage bucket `business-files-private`
- Image upload into public bucket `business-images-public`
- Interested Investors tab; business can approve connection
- Request Data tab; business can mark uploaded/fulfilled
- VCPC support box in left dashboard menu

### Investor flow

- Investor registration/dashboard
- Default investor dashboard language is English
- Saved Businesses tab
- Investment Criteria tab with sector/revenue/EBITDA inputs and recommended businesses grid
- Proposals tab showing business name, investment ask/stake, EBITDA, Business Quality Score and approve/request/decline actions
- Privacy settings for email and WhatsApp/Zalo phone, with country calling-code table for all countries

### Admin flow

- Admin login via Supabase Auth
- Admin can edit all 6 seeded deals and any business created from database
- Admin can edit imported investors
- Admin show/hide businesses/investors; public listings update accordingly
- Admin creates promo codes, including free Business/Investor promo until 10/7/2026 with quota limits
- Admin edits Business Quality Score criteria/weights
- Admin approves business pending changes
- Admin manages Request Data queue

## Setup

1. Create Supabase project.
2. In Supabase SQL Editor, run migrations in order:
   - `supabase/migrations/0001_schema.sql`
   - `supabase/migrations/0002_country_calling_codes_seed.sql`
   - `supabase/migrations/0003_quality_criteria_seed.sql`
3. Create `.env` from `.env.example`.
4. Run:

```bash
npm install
npm run seed
npm run build
```

5. Deploy to Netlify:
   - Build command: `npm run build`
   - Publish directory: `dist`
   - Env vars: `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`, `VITE_SITE_URL=https://deals68.com`
6. Point domain `deals68.com` to Netlify DNS.

## Important production notes

- `SUPABASE_SERVICE_ROLE_KEY` is only for local seed scripts. Never expose it in Netlify public environment variables.
- `profiles.initial_password` exists only because Beta requires Admin to view generated account passwords. Remove/rotate before production public onboarding.
- Payment is prepared for QR/manual approval. Senpay/BaoKim, Stripe and PayPal webhook handlers are placeholders to implement next.
- Email alerts are data-ready but not wired to an email provider yet.
- Multi-country expansion is supported by `country_calling_codes`, profile country/timezone/language fields and translatable VI/EN fields. Additional languages can be added by adding localized columns or a translation table.
